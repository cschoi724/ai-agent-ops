# AI Document Governance

작성일: 2026-06-29  
상태: Draft v1  
범위: `.ai/` 운영 문서 보호 규칙

## 1. 목적

`.ai/` 아래 문서는 AI 에이전트 운영 체계의 기준 문서다. PM Agent, Dev Agent, QA Agent를 포함한 어떤 Agent도 사용자 승인 없이 이 문서들을 임의로 수정하면 안 된다.

이 문서는 운영 문서의 변경 권한, 변경 절차, 예외 조건을 정의한다.

프로젝트별 작업 보드, handoff, 보고서, Agent 운영 결정 기록은 `.ai/`가 아니라 `.ai_project/`에 둔다. 제품/기술 결정은 적용 대상 프로젝트 문서 영역에 둔다. `.ai_project/`는 프로젝트 종속 문서 영역이며, 수정 규칙은 `.ai/project_workspace.md`를 따른다.

## 3. 변경 권한

사용자만 변경을 승인할 수 있다.

Agent 역할별 권한:

| 역할 | 승인 없는 수정 | 승인 후 수정 | 비고 |
|---|---:|---:|---|
| PM Agent | 금지 | 가능 | 변경안 제안은 가능 |
| Dev Agent | 금지 | 원칙적으로 불가 | 개발 작업 중 운영 문서 수정 금지 |
| QA Agent | 금지 | 원칙적으로 불가 | 검토 의견 작성만 가능 |
| 기타 Agent | 금지 | 사용자 지정 시 가능 | 역할과 범위가 명확해야 함 |

## 4. 허용되는 작업

사용자 승인 없이 가능한 작업:

- `.ai/` 문서 읽기
- `.ai/` 문서 내용 요약
- 변경 필요성 제안
- 변경안 초안 작성
- 별도 메시지로 패치 제안

사용자 승인 후 가능한 작업:

- `.ai/` 운영 문서 직접 수정
- `.ai/` 운영 문서 추가
- `.ai/` 운영 문서 삭제
- 보호 대상 문서의 구조 변경
- `ai-agent-ops` 템플릿 저장소에 반영할 문서 초안 생성
- 현재 프로젝트에 `.ai_project/` 초기 구조 생성

## 5. 변경 절차

1. Agent는 변경 필요성을 먼저 설명한다.
2. 변경 대상 파일을 명시한다.
3. 변경 이유와 영향을 받는 Agent를 정리한다.
4. 사용자에게 승인 여부를 묻는다.
5. 사용자가 승인한 범위 안에서만 수정한다.
6. 수정 후 변경 파일과 변경 내용을 보고한다.

## 6. 금지사항

- 사용자 승인 없이 `.ai/` 운영 문서를 수정하지 않는다.
- 프로젝트별 협업 기록을 `.ai/`에 누적하지 않는다.
- PM Agent가 자신의 판단만으로 운영 규칙을 바꾸지 않는다.
- Dev Agent가 구현 편의를 위해 운영 규칙을 바꾸지 않는다.
- QA Agent가 검증 기준을 통과시키기 위해 운영 규칙을 완화하지 않는다.
- 기존 보호 문서를 덮어쓰지 않는다.
- 논의 중인 아이디어를 확정 규칙처럼 문서에 반영하지 않는다.

## 7. 예외

아래 경우에도 사용자 승인 후 진행한다.

- 오탈자 수정
- 링크 경로 수정
- 날짜 갱신
- 문서 이동
- 템플릿 문구 정리

단, 사용자가 한 메시지 안에서 명시적으로 수정 지시를 준 경우에는 그 범위 안에서 승인된 것으로 본다.

## 8. 권장 운영 방식

Agent가 `.ai/` 문서 변경이 필요하다고 판단하면, 직접 수정하지 않고 아래 형식으로 먼저 제안한다.

```text
변경 제안:
- 대상 문서:
- 변경 이유:
- 변경 내용 요약:
- 영향 받는 Agent:
- 승인 요청:
```

사용자가 승인하면 그때 문서를 수정한다.
