# Policies

`policies/`는 운영 중 지켜야 하는 정책과 보호 규칙을 담는다.

| 문서 | 역할 |
|---|---|
| `ownership_model.md` | path / domain / document ownership |
| `coordination_policy.md` | dependency, blocked, rework, 병렬 조율 |
| `board_model.md` | project board / team board 기준 |
| `branch_pr_policy.md` | branch / commit / PR / merge 정책 |
| `commit_policy.md` | commit 권한과 제한 |
| `document_governance.md` | 운영 문서 보호 기준 |
| `knowledge_policy.md` | `.ai_knowledge/` 갱신과 source of truth 규칙 |
| `migration_policy.md` | 기존 프로젝트의 운영모델 마이그레이션 기준 |
| `project_workspace.md` | `.ai_project/` 구조와 수정 권한 |
| `update_policy.md` | 운영체계 갱신 정책 |
| `versioning_policy.md` | core version 기록과 update risk 기준 |
