# AI Agent Commit Policy

작성일: 2026-06-29  
상태: Draft vNext  
범위: AI Agent 운영 문서와 적용 대상 프로젝트 변경의 커밋 분리 기준

## 1. 목적

이 문서는 AI Agent 운영 템플릿, 프로젝트별 Agent 워크스페이스, 실제 프로젝트 코드/문서 변경을 커밋 단위에서 분리하는 기준을 정의한다.

## 2. 경로별 커밋 소유권

| 경로 | 성격 | 기본 커밋 소유권 |
|---|---|---|
| `.ai/` | AI Agent 운영 템플릿 저장소 | `ai-agent-ops` 저장소 |
| `.ai_project/` | 적용 대상 프로젝트의 Agent 협업 상태 | 적용 대상 프로젝트 저장소 |
| 프로젝트 코드/문서 | 실제 제품 구현과 프로젝트 문서 | 적용 대상 프로젝트 저장소 |

## 3. 기본 원칙

- `.ai/` 변경은 `ai-agent-ops` 템플릿 저장소 변경으로만 취급한다.
- 적용 대상 프로젝트의 코드/문서 커밋에 `.ai/` 변경을 섞지 않는다.
- `.ai_project/`는 프로젝트별 Agent 운영 상태이므로 기본적으로 적용 대상 프로젝트 저장소에 포함한다.
- 단, 프로젝트 초기 마이그레이션이나 운영 실험 단계에서는 사용자 결정으로 `.ai_project/`를 로컬 전용으로 둘 수 있다.
- `.ai_project/`를 로컬 전용으로 둘 경우 사유, 재검토 조건, 향후 저장소 포함 시점을 migration 문서 또는 `.ai_project/ops_decisions.md`에 기록한다.
- 프로젝트 코드/문서 변경과 `.ai/` 템플릿 변경은 항상 별도 커밋으로 분리한다.
- 커밋, push, 배포, 외부 설정 변경은 사용자 승인 후 진행한다.
- 프로젝트가 `.ai/policies/branch_pr_policy.md` 기준으로 branch/PR 전략을 명시한 경우, task branch 안의 commit/push/PR 권한은 해당 프로젝트 설정을 따른다.
- 어떤 설정에서도 default branch 직접 push는 금지한다.

## 4. 커밋 분리 기준

| 변경 조합 | 같은 커밋 허용 여부 | 기준 |
|---|---|---|
| `.ai/` + 프로젝트 코드 | 금지 | 템플릿 저장소와 프로젝트 저장소가 다름 |
| `.ai/` + 프로젝트 문서 | 금지 | 템플릿 변경과 프로젝트 변경은 분리 |
| `.ai/` + `.ai_project/` | 금지 | 운영 템플릿과 프로젝트 운영 상태는 분리 |
| `.ai_project/` + 프로젝트 문서 | 프로젝트 정책에 따름 | 같은 작업의 운영 기록과 문서 변경이면 허용 가능 |
| 프로젝트 코드 + 관련 프로젝트 문서 | 허용 가능 | 같은 작업 결과라면 같은 커밋 가능 |

## 5. 적용 대상 프로젝트 정책

새 프로젝트에 AI Agent 운영을 적용할 때 Lead Role 또는 Ops Governance Role은 아래를 먼저 확인한다.

1. `.ai/`가 별도 저장소로 관리되는지 확인한다.
2. 적용 대상 프로젝트 `.gitignore`에서 `.ai/`가 제외되는지 확인한다.
3. `.ai_project/`를 기본값대로 프로젝트 저장소에 포함할지, 초기 예외로 로컬 전용으로 둘지 사용자에게 확인한다.
4. `.ai_project/`를 프로젝트 저장소에 포함할 경우 `.gitignore`에서 제외하지 않는다.
5. 커밋 전 변경 경로가 어떤 저장소 소유인지 확인한다.
